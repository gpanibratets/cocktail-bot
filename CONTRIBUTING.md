# Правила оформления изменений и pull request'ов

## Ветки

- Для любых изменений создаём отдельную ветку от `main`.
- Формат имени ветки:
  - `feature/<краткое-описание>` — новые возможности;
  - `fix/<краткое-описание>` — исправления багов;
  - `chore/<краткое-описание>` — инфраструктура, зависимости и прочее.

Примеры:

- `feature/analytics`
- `fix/random-command-error`

## Коммиты

- Коммиты делаем **логическими и небольшими**.
- Сообщение коммита на английском в формате:
  - `add ...` — добавление новой функциональности;
  - `update ...` — доработка существующей;
  - `fix ...` — исправление ошибки;
  - `chore ...` — технические изменения.

Примеры:

- `add basic analytics storage`
- `fix random command error handling`

## Pull request'ы

- Один PR — одна логическая задача.
- Название PR на английском, в стиле:
  - `Add basic analytics for user events`
- Описание PR (минимум):
  - **Summary** — что сделано (1–3 пункта);
  - **Motivation** — зачем это нужно;
  - **Testing** — как проверяли (команды / сценарии).

Шаблон описания PR:

```text
## Summary
- ...

## Motivation
- ...

## Testing
- [ ] Manual: ...
- [ ] Other: ...
```
